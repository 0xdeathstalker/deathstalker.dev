# The Sticky Footer Reveal Effect

When you scroll to the bottom of a page and the footer slides into view like it was always sitting underneath the content, you are looking at the sticky footer reveal effect. The page content lifts off the footer as you scroll, revealing it from below. This post explains how it works and how to build it yourself.

---

## What It Looks Like

```
BEFORE SCROLLING                  WHILE SCROLLING                   FULLY REVEALED

┌──────────────┐ ← top            ┌──────────────┐ ← top            ┌──────────────┐
│              │                  │              │                  │              │
│   Page       │                  │   Page       │                  │              │
│   Content    │                  │   Content    │                  │   Footer     │
│              │                  │   (ending)   │                  │   (fully     │
│              │                  │              │                  │    visible)  │
└──────────────┘ ← bottom         ├──────────────┤                  └──────────────┘
                                  │   Footer     │ ← peeking
[footer hidden below viewport]    │   (partial)  │
                                  └──────────────┘
```

The footer is not fixed to the screen. It sits below the page content in the document and gets revealed as the user scrolls, almost like lifting a sheet of paper off a table to reveal what is underneath.

---

## How It Works

There are three nested elements, each playing a specific role.

### Layer 1: The outer footer element

```
┌────────────────────────────────────────────────┐
│  <footer>  height: 750px  (document space)     │
│                                                │
│  This is the only space the footer takes up    │
│  in the normal document flow. Think of it      │
│  as a "placeholder" that reserves room.        │
└────────────────────────────────────────────────┘
```

The outer element has a fixed height matching the footer's visual height. This tells the browser "reserve this much space at the bottom of the page." A `clipPath` is added to ensure nothing visually bleeds outside this boundary during the animation.

### Layer 2: The tall inner container

```
                                  ← top of footer's document position
┌────────────────────────────────┐
│  -top: 100vh  (pulled up)      │  ← starts 100vh above the footer
│                                │
│                                │
│  height: 100vh + 750px         │  ← total height gives scroll room
│                                │
│                                │
└────────────────────────────────┘
                                  ← bottom aligns with footer bottom
```

This div is made taller than the viewport and pulled upward by exactly one viewport height. The result is that this container spans from one viewport height above the footer's top all the way to the footer's bottom. This extra height is the "scroll budget" that the sticky child uses to travel.

### Layer 3: The sticky content div

```
SCROLL POSITION A                 SCROLL POSITION B

┌──── tall container ────────┐    ┌──── tall container ────────┐
│                            │    │                            │
│                            │    │  ┌──── sticky div ──────┐  │
│                            │    │  │  Footer Content      │  │  ← stuck at
│                            │    │  │                      │  │    top: 100vh - 750px
│  ┌──── sticky div ──────┐  │    │  └──────────────────────┘  │
│  │  Footer Content      │  │    │                            │
│  └──────────────────────┘  │    │                            │
└────────────────────────────┘    └────────────────────────────┘
```

The sticky div has `top: calc(100vh - 750px)`. This means it sticks at a position that places its bottom edge exactly at the bottom of the viewport. It starts scrolling when the tall container enters the viewport and stops when the container ends, at which point the footer is fully visible.

### Full picture

```
Document Layout (zoomed out)

┌─────────────────────┐  ↑
│                     │  │
│    Page Content     │  │  normal page height
│                     │  │
│                     │  ↓
├─────────────────────┤  ← footer top (in document flow)
│  ┌───────────────┐  │  ↑
│  │  tall inner   │  │  │  100vh + 750px
│  │  container    │  │  │
│  │               │  │  │
│  │  ┌─────────┐  │  │  ↓
│  │  │ sticky  │  │  │  ← sticks here once it reaches top: (100vh - 750px)
│  │  │ content │  │  │
│  │  └─────────┘  │  │
│  └───────────────┘  │
└─────────────────────┘  ← footer bottom (750px below footer top)
```

## Steps to Recreate It

**Step 1: Decide your footer height.**

Pick a fixed pixel height for your footer. In this example it is `750px`. All the calculations are based on this number.

**Step 2: Create the outer footer element.**

Give it a `relative` position and set its height to your chosen footer height. Add a `clipPath` covering the full rectangle so that nothing overflows visually during the scroll animation.

```css
height: 750px;
position: relative;
clip-path: polygon(0% 0, 100% 0%, 100% 100%, 0 100%);
```

**Step 3: Create the tall inner container.**

Inside the footer, add a div with:
- height equal to `100vh + footer height`
- negative top offset equal to `-100vh`
- `position: relative`

This pulls the container up so it starts one viewport height before the footer's document position and gives the sticky child enough room to scroll.

```css
height: calc(100vh + 750px);
top: -100vh;
position: relative;
```

**Step 4: Create the sticky content wrapper.**

Inside the tall container, add the div that will hold your actual footer content. Make it:
- the same height as the footer (`750px`)
- `position: sticky`
- `top` set to `calc(100vh - 750px)`

This value places the sticky element so that it sits flush with the bottom of the viewport when it sticks.

```css
height: 750px;
position: sticky;
top: calc(100vh - 750px);
```

**Step 5: Add your footer content inside the sticky wrapper.**

Everything inside the sticky wrapper is your visible footer: logo, links, copyright text and so on.

## Why Not Just Use `position: fixed`?

A fixed footer is always on screen. This effect only shows the footer when you reach the end of the page. It also allows the footer to participate in normal document flow so the page does not have awkward empty space at the bottom. The sticky approach gives you a smooth, scroll-driven reveal without any JavaScript.

## Quick Reference

| Property | Element | Value | Purpose |
|---|---|---|---|
| `height` | outer footer | `750px` | Reserves document space |
| `clip-path` | outer footer | full rect | Prevents visual overflow |
| `height` | inner container | `100vh + 750px` | Scroll budget for sticky child |
| `top` | inner container | `-100vh` | Pulls container up to start early |
| `position` | inner container | `relative` | Establishes containing block |
| `height` | sticky div | `750px` | Matches visual footer height |
| `position` | sticky div | `sticky` | Enables sticky behavior |
| `top` | sticky div | `100vh - 750px` | Aligns bottom edge to viewport bottom |
